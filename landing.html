Customer Churn Analysis

The Power bi being a business intelligence tool was use in this project to get data from excel flat file and excel being a data storage tool could only store up to 1 million row level, then ETL, modelling, and visualisating processes was performed using Power BI as well. DAX (data analyst expression) was also utilise in this project and One drive was also utilised in this project to optimise data integration and other benefit like auto update when there is any change in the data. The aim of this projects is to use all the tools involve to gain insights about churn and un-churn customers and churn and un-churn customers service such like impact of churn customers on services.

Stores Sales and Product Analysis

In this project, Power BI was leveraged not only to extract data but also to conduct the ETL (Extract, Transform, Load) processes. The data originated from an Excel workbook composed of three spreadsheet tables. Additionally, OneDrive was utilized to facilitate seamless data integration and automatic updates, enhancing our ability to gain timely insights into store and product sales performance. The data modeling phase involved creating star schemas to establish relationships between tables, which helped streamline our analysis. Furthermore, DAX (Data Analysis Expressions) was employed to develop a comprehensive date table, crucial for detailed time-series analysis. This setup enabled a thorough exploration of sales trends and performance metrics across multiple dimensions

Hola Cookies Sales Analysis

In this project, a Microsoft Access Database was utilized, providing a more robust infrastructure compared to traditional Excel setups. This choice eliminated the need for extensive data modeling within Power BI and the necessity to store flat files in OneDrive for data integration optimization. Power BI was employed to connect directly to the Hola Cookies Company dataset housed in the Microsoft Access database, necessitating only minimal ETL processes. Nevertheless, DAX (Data Analysis Expressions) played a crucial role in creating a date table, along with calculated columns and aggregation measures, to enrich the analysis. This setup streamlined the workflow and enhanced our ability to extract actionable insights efficiently.
